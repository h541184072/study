1.什么是设计模式

```text
1.概念
使用设计模式是为了重用代码、让代码更容易被他人理解、保证代码可靠性。
2.设计原则
单一职责原则

    一个程序只做好一件事
    如果功能过于复杂就拆分开，每个部分保持独立
最少知识原则( 中介者模式/外观模式)

    应当尽量减少对象之间的交互

开放/封闭原则(after,before)

    对扩展开放，对修改封闭
    增加需求时，扩展新代码，而非修改已有代码

里氏替换原则

    子类能覆盖父类
    父类能出现的地方子类就能出现

接口隔离原则

    保持接口的单一独立
    类似单一职责原则，这里更关注接口

依赖倒转原则

    面向接口编程，依赖于抽象而不依赖于具
    使用方只关注接口而不关注具体类的实现

```
